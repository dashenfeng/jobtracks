# 模块一：投递管理系统

## 状态流转
状态列表：待投递 → 已投递 → 笔试 → 一面 → 二面 → 三面 → HR面 → Offer / 挂 / 放弃
说明：放弃为终止状态，用于用户主动放弃的场景（如时间冲突、价值观不符等）。每条记录可附带备注（碎碎念）。

## 字段需求
- 公司名称（必填）
- 岗位名称（必填）
- 岗位链接（可选）
- 城市（可选）
- 投递渠道（BOSS/牛客/官网/内推/其他）
- 当前状态（枚举）
- 薪资范围（可选，字符串或区间）
- 最近更新时间（自动）
- 备注（可选，自由文本）
- 附件（可选，支持上传 JD 截图、面经文档等）

## 一键分析
输出形式：文字描述为主 + 图表为辅，通过 Tab 栏切换
分析维度：
- 渠道效果（各渠道投递量、回复率、面试率）
- 阶段转化率（各阶段通过率）
- 时间规律（投递时间 vs 回复率）
- 城市/公司偏好（各城市面试机会分布）

## 权限模型
支持四种级别（可单选或多选组合）：
- 只读：可查看所有记录，不可修改
- 可评论：可添加备注，不可修改状态
- 可编辑：可更新状态、添加记录
- 字段级控制：可限制特定字段不可见（如薪资）

## 导入导出
- 导入：支持 .csv / .xlsx 格式，用户先下载模板 → 填写 → 上传 → 预览确认 → 批量入库（新增，不覆盖）
- 导出：支持导出为 Excel / CSV，可选导出全部或筛选后的数据

# 模块二：JSON 工具 + 驼峰转换

## 功能范围（单页多标签，Tab 切换）
1. **JSON 格式化 / 压缩**：美化显示或压缩为一行
2. **命名风格转换**：支持 snake_case ↔ camelCase、snake_case ↔ PascalCase、kebab-case ↔ camelCase，支持全选互转
3. **JSON 对比**：两份 JSON 的差异高亮显示
4. **JSON 路径提取**：输入 JSONPath 表达式（如 $.data.list[0].name），提取对应值

## 交互设计
- **命名风格转换 Tab**：页面分为左右两个大型编辑框
  - 左侧：源输入区（用户粘贴或上传 .json 文件）
  - 右侧：转换结果区（实时显示转换后的 JSON）
  - 转换结果区右上方提供：复制按钮（复制到剪贴板）、下载按钮（下载为 .json 文件）
- **其他 Tab**：根据功能特性设计合适的交互布局

## 特殊功能
- **excludeKey**：用户可填写一个或多个 key 名称，这些 key 在命名风格转换时保持不变（不参与转换）
- **格式保留**：转换命名风格时，JSON 原有的缩进、换行等格式保持不变

## 数据输入
- 手动粘贴到编辑器
- 上传 .json 文件
- URL 拉取（Phase 2 再考虑）

## 数据输出
- 替换编辑器内容（原地生效）
- 复制到剪贴板
- 下载为 .json 文件

# 模块三：环境变量保险箱（EnvVault）

## 存储内容
每条记录包含以下字段：
- Key 名称（如 `OPENAI_API_KEY`）
- Value（支持多行文本，如 SSH 私钥、恢复码等）
- 标签（一个记录可有多个标签，用于分组和筛选）
- 备注（可选，自由文本）

## 分组与筛选
- 按标签分组，支持按标签筛选
- 每个用户只能看到自己创建的记录（数据按用户 ID 隔离）

## 查看权限与安全
- 列表中默认显示掩码（如 `sk-****abcd`）
- 点击“显示”按钮后明文展示，同时记录审计日志
- 提供“复制”按钮，复制明文到剪贴板（复制操作也记录审计日志）

## 审计日志
- 记录：谁在什么时间查看了哪条记录的明文（包括查看和复制操作）
- 保存时长：180 天
- 密钥被查看时，向用户发送站内通知（右上角消息中心）

## 操作功能
- 增删改查（CRUD）
- 一键复制 Value 到剪贴板
- 批量导入：支持从 .env 文件导入（解析 KEY=VALUE 格式）
- 批量导出：支持导出为 .env 文件
- 密钥轮换：生成新值，旧值归档（仍可在审计日志中追溯，但不再作为当前值展示）

## 与本项目其他模块的关联
- 暂不关联其他模块，独立运行

# 模块四：接口快照对比（Snapshot Diff）

## 快照来源
- 手动粘贴接口返回的 JSON / XML / 文本
- 上传文件（如 .json 响应文件）
- URL 请求捕获（Phase 2 再考虑）

## 快照组织与管理
- 每条快照包含：快照名称（必填）、备注（可选）、标签（可选，支持多标签）、所属项目（可选）、创建时间（自动）
- 支持按时间线浏览（所有快照按创建时间倒序）
- 支持按标签 / 项目筛选
- 支持标记某个快照为“基线版本”，后续对比时默认以此版本为基准

## 对比方式
支持两种对比模式，用户可切换：
1. **左右并排对比**：左侧旧快照 / 基线版本，右侧新快照，差异高亮显示
2. **行级 Git Diff 对比**：只显示有差异的行，相同行折叠，类似 GitHub PR 的 diff 视图

## 附加功能（当前实现）
- 快照命名
- 标记基线版本

## 附加功能（后续扩展，仅列出不实现）
- 导出对比报告（HTML / PDF）
- 自动定时抓取（定期请求接口并保存快照）

## 与本项目其他模块的关联
- 暂不关联其他模块，独立运行

## 记录组织方式
- 按版本号（如 v0.1.0、v0.2.0） + 发布时间共同标识一条记录

## 展示形式
- 纵向时间线布局，每个版本一张卡片
- 卡片内展示该版本的详细变更内容

## 与其他模块的关联

- 具体触发时机和粒度由后续实现决定

## 权限
- Changelog 页面对所有被邀请的用户可见（只读）

# 模块六：AI Agent 调度层（Phase 2）

## 核心定位
作为整个工具库的智能调度层，通过自然语言交互，让用户以对话方式调用各工具功能，特别是处理跨工具组合操作和写死逻辑覆盖不了的长尾需求。

## 功能范围
- **导航**：用户说出页面名称，Agent 跳转到对应工具页
- **单工具操作**：在当前页上下文内执行单一工具功能（如“把这段 JSON 转成小驼峰”）
- **跨工具组合操作**：串联多个工具功能（如“把这两个快照都转成驼峰，然后对比差异”）
- **查询与分析**：调用投递分析引擎等模块，返回分析结果

## 对话形态
- 支持单轮指令（简单任务，一次性执行并返回）
- 支持多轮连续对话（复杂任务，用户可追问、修正，Agent 保持上下文记忆）

## 触发方式
- 全局快捷键（Cmd+K / Ctrl+K）
- 页面右下角悬浮按钮
- 两者同时支持

## 成本控制策略
- 优先使用免费或极低成本模型（如 DeepSeek）
- 对于确定性操作（如驼峰转换），优先用规则匹配 + 少量 LLM 调用，减少不必要的 API 消耗
- 设定每月预算上限（具体金额由后续实现决定）

## 用户偏好记忆
- Agent 可记住用户的常用操作模式和偏好（如频繁使用的功能、常用的命名风格等）
- 每次对话开始时加载用户偏好作为上下文，提升交互效率
- 长期记忆存储在数据库中，按用户 ID 隔离
